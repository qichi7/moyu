# 吃豆人游戏

一个使用HTML5 Canvas和JavaScript实现的经典吃豆人游戏。

## 功能特性

- 🎮 经典吃豆人游戏玩法
- 🗺️ 精心设计的游戏地图
- 👻 可自定义数量的幽灵敌人（1-10个）
- 🏆 分数系统和最高分记录
- 📱 响应式设计
- 🎨 现代化UI界面
- 🖼️ 完全自定义图片（吃豆人、幽灵、豆子、胜利图片）
- 🎵 搞怪配乐系统
- 🚫 不能反向移动
- 🔄 触墙自动反弹
- 📊 GitHub Gist 排行榜

## 如何开始

1. 在浏览器中打开 `index.html` 文件
2. 点击"开始游戏"按钮
3. 使用方向键控制吃豆人移动

## 游戏规则

- 吃掉所有豆子即可获胜
- 避免被幽灵抓住
- 每个豆子得10分
- 最高分会自动保存到本地存储
- 吃豆人不能直接反向移动（如从右向左移动时不能立即向左）
- 当吃豆人碰到墙壁时会自动反弹到其他可用方向

## 排行榜系统

### ⚠️ 重要说明

本游戏使用 **GitHub Gist** 存储排行榜数据。由于 Gist ID 已硬编码为作者专属，排行榜功能有以下限制：

| 功能 | 说明 |
|------|------|
| **查看排行榜** | 所有玩家都可以查看 |
| **保存成绩** | **只有 Gist 所有者（作者）可用** |

### 为什么其他玩家无法保存？

GitHub Token 只能写入自己账号下的 Gist。硬编码的 Gist 属于 `qichi7`，其他玩家的 Token 无写入权限（会返回 403）。

### 如果你想有自己的排行榜？

1. Fork 本仓库
2. 创建自己的 GitHub Gist：https://gist.github.com/
3. 修改 `game.js` 中的 `HARDCODED_GIST_ID` 为你的 Gist ID
4. 创建 Personal Access Token（需要 `gist` 权限）

### Token 安全

- Token 使用 **sessionStorage** 存储，关闭浏览器后自动清除
- Token 不会被持久化保存，每次新会话需要重新输入

## 安全特性

本游戏已实现以下安全防护：

| 安全措施 | 说明 |
|----------|------|
| **XSS防护** | 排行榜数据使用 `textContent` 渲染，不使用 `innerHTML` |
| **数据校验** | 从 Gist 获取的数据经过白名单验证 |
| **Token保护** | Token 存储在 `sessionStorage`，关闭浏览器即失效 |

## 技术栈

- HTML5 Canvas
- 原生JavaScript
- CSS3动画和渐变
- Web Audio API（音效系统）
- GitHub Gist API（排行榜）

## 地图设计

游戏使用15x19的网格地图，包含：
- 墙壁（1）
- 通道（0）
- 豆子位置
- 幽灵出生点

## 控制方式

- ⬆️ 上箭头：向上移动
- ⬇️ 下箭头：向下移动
- ⬅️ 左箭头：向左移动
- ➡️ 右箭头：向右移动
- 空格键：开始/暂停游戏

## 自定义功能

### 图片自定义
- **吃豆人图片**：上传张嘴和闭嘴两种状态图片
- **幽灵图片**：支持上传多个图片，按编号顺序替换幽灵
- **豆子图片**：自定义豆子外观
- **胜利图片**：自定义游戏胜利时的结算图片

### 音效系统
- **启用/禁用音效**：通过复选框控制
- **音量调节**：通过滑块调节音量（0-100%）
- **音效类型**：
  - 吃豆子：清脆的"叮"声
  - 触墙反弹：低沉的"咚"声
  - 游戏胜利：欢快的胜利旋律
  - 游戏失败：悲伤的下降音效

## 新增功能说明

### 1. 不能反向移动
吃豆人不能直接掉头移动，比如正在向右移动时，不能立即向左移动。这增加了游戏的策略性和挑战性。

### 2. 触墙反弹
当吃豆人碰到墙壁时，会自动反弹到其他可用的方向，而不是停在原地。这确保了游戏流畅性，避免了被卡死的情况。

### 3. 胜利结算图片
玩家可以上传自定义的胜利结算图片，在游戏胜利时显示个性化的庆祝画面。

### 4. 搞怪配乐
使用Web Audio API实现了有趣的音效系统：
- 吃豆子时有清脆的提示音
- 碰到墙壁时有搞怪的反弹音效
- 胞利时有欢快的旋律
- 失败时有悲伤的音效
- 所有音效都可以调节音量
