# 中午吃点啥 - 任务规划

---

## 项目概述

一个午餐选择轮盘抽奖网页，帮助用户随机决定中午吃什么。

---

## 功能需求

### 1. 轮盘主体
- 圆形轮盘，显示所有可选菜品选项
- 中间有"我要吃"按钮
- 点击按钮后指针开始旋转
- 最终指向一个选项，放大显示结果

### 2. 选项管理
- 如果没有选项 → 显示"你没饭吃了"
- 可以添加新选项（需要输入Token）
- 选项存储在GitHub Gist

### 3. 历史记录
- 记录最近三次吃的选项
- 这三个选项暂时从轮盘中移除
- 新记录覆盖旧记录，被覆盖的选项重新加入轮盘

## 技术架构

### 目录结构
```
中午吃点啥/
├── index.html          # 主页面
├── style.css           # 样式文件
├── wheel.js            # 轮盘逻辑
├── gist-manager.js     # Gist数据管理
└── PLAN.md             # 任务规划（本文件）
```

### 数据存储（GitHub Gist）
```json
{
  "options": [
    "麻辣烫",
    "黄焖鸡",
    "盖浇饭",
    "面条",
    "饺子"
  ],
  "history": [
    { "name": "麻辣烫", "time": "2026-04-28 12:00" },
    { "name": "黄焖鸡", "time": "2026-04-27 12:00" },
    { "name": "盖浇饭", "time": "2026-04-26 12:00" }
  ]
}
```

### Token处理
- **不硬编码Token**
- 添加新选项时弹出输入框
- Token仅在写入Gist时使用
- 可选：使用sessionStorage临时存储

## 注意事项

- Token **绝不硬编码**，用户每次添加选项时输入
- 历史记录只保留最近3条
- 被移除的选项显示为灰色或不可点击
- 轮盘至少要有可旋转的选项才能工作
- 空选项时显示"你没饭吃了"并提示添加

---

## Gist创建步骤

1. 访问 https://gist.github.com/
2. 创建新Gist，文件名：`lunch-options.json`
3. 内容：`{"options": [], "history": []}`
4. 获取Gist ID
5. 将Gist ID硬编码到代码中（Token不硬编码）
